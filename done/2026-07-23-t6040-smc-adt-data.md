## `/arm-io/smc` (compatible `iop,ascwrap-v6`, interrupt-parent = AIC)

DT reg mapping is the proven ADT→DT split (`0xABCDEFGHI` → `<0xA 0xBCDEFGHI>`,
same as dockchannel `0x50880c000`→`<0x5 0x0880c000>`):

| ADT reg | DT `reg` cell | size | likely role |
|---|---|---|---|
| `0x30c600000` | `<0x3 0x0c600000 …>` | `0x88000` | ASC wrapper (mailbox + SMC core region) |
| `0x30c050000` | `<0x3 0x0c050000 …>` | `0x4000` | secondary control window |
| `0x30c810000` | `<0x3 0x0c810000 …>` | `0x100` | small control/SRAM-index window |

ASC mailbox IRQs (AIC inputs, the four ASC doorbells in order): **997, 996,
999, 998** (send-empty, send-not-empty, recv-empty, recv-not-empty). Single
child node `iop-smc-nub`.

## Remaining sub-task (DT authoring)

The upstream macsmc DT (t8112/t600x) uses a **separate `apple,asc-mailbox`
node** + an `smc` node with `reg-names = "smc","sram"` (smc 0x4000, sram
0x100000) and `gpio`/`rtc`/`reboot`/nvmem children. The J614s ascwrap-v6 exposes
three windows and no obvious 0x100000 SRAM, so before authoring:

1. Read `/arm-io/smc/iop-smc-nub` (+ any `smc-sram`/`smc-pmu` grandchildren) for
   the SRAM window and the nvram cell offsets (reboot/rtc) — parse from the raw
   ADT (`adt.py`), not `ipsw --json` (it mangles blobs, cf. cpufreq/DVFM).
2. Attribute the three ascwrap windows to Linux `mailbox` vs `smc` vs `sram`
   using the `apple,asc-mailbox-v4`/`apple,smc` reg convention (the mailbox reg
   is a fixed sub-offset of the ascwrap base).
3. Author the mailbox + smc node with the four IRQs above, `gpio`/`rtc`/`reboot`
   children (read-only status keys only — battery gauge, power button, lid);
   **no charger-control or PMU/SPMI writes** (the `smc-pmu`-backed GPIOs stay
   forbidden, per the trackpad audit). Then dtbs_check + build + pin.
