# J614s BCM4388 WiFi/BT firmware extraction (ticket 014)

Ticket 014 is complete offline. The paired `apple,mriya` wireless firmware is
staged on the host for initramfs/rootfs use, with full provenance and hashes.
No rig run is proposed: the BCM4388 sits on PCIe port 0 behind the unresolved
operation-115/shared-PHY blocker (NEXT_STEPS §0.1), and neither `brcmfmac` nor
`hci_bcm4377` is in the current kernel config.

  `a6e764ca158e10ea2ace9b74701f445eefbf012c9cdb5aaa616aa10a0b5197ef`) names
  both that kernelcache path and the member extracted here.

## Board/chip identification

The saved J614s ADT gives `/product/wifi-chipset = "4388"` and
`/arm-io/wlan/module-instance = "mriya"`
(`done/2026-07-14-t6040-wireless-pcie-map.md`; PCI `14e4:4434` WiFi,
`14e4:5f72` BT on port 0). The 25F84 Bluetooth tree carries `Mriya` firmware
only for stepping **C2** (`BCM4388C2_23.5.224.14xx_..._Mriya_...`), so the
fitted module is 4388 C2. WiFi firmware is staged for both C0 and C2 anyway;
`brcmfmac` selects by the chip's runtime revision register.

## macOS 26.x layout changes (input for ticket 026)

The stock asahi-installer flow (commit `c53d66dc7193`-era `asahi_firmware`)
cannot process a 26.x image unmodified. Everything found:

1. **BaseSystem is AEA-wrapped** (`.dmg.aea`); the installer has no AEA path.
2. **WiFi payloads moved into the AppleBCMWLAN dext.** The legacy
   `usr/share/firmware/wifi/` tree still exists but holds only
   `*-PlatformConfig.plist` files and now-dangling symlink stubs (e.g.
   `C-4388__s-C2/aruba.trx -> paradise.trx` with no `paradise.trx` anywhere).
   The real tree, same `C-<chip>__s-<step>` structure and zero dangling links,
   is at
   `/System/Library/DriverExtensions/com.apple.DriverKit-AppleBCMWLAN.dext/Firmware/`.
   `WiFiFWCollection` run against that tree works, except:
3. **New filename forms crash the wifi parser**: an `F-` dimension
   (`P-mahoganya_M-DRGN_V-g_F-bb__m-1.1.txt`) and `*_gen*.clmb` variants —
   86 non-mriya files, excluded here (all are 4388 C0/C2 `mahoganya`/`gen`
   entries; list preserved in the regeneration recipe's exclusion globs).
4. **New WiFi blob types** `.pcfb` and `.man` are not in the collector's
   extension map. The mriya `.pcfb` blobs are staged raw (below); no Linux
   consumer exists today.
5. **New Bluetooth vendor `AMKOR`** is missing from `VENDORMAP` (MUR/USI/GEN),
   so the stock collector emits only the USI mriya BT variant and warns on the
   AMKOR one. Both raw AMKOR mriya files are staged (below). If the fitted
   module turns out AMKOR-sourced, `hci_bcm4377` firmware naming needs a
   matching extension — flag for ticket 028/037 once OTP is readable.
6. The BT tree also now contains MediaTek `MT7932*` firmware (skipped, warns).

## Initramfs wiring

`scripts/t6040-make-initramfs.sh` gained a `VENDORFW_DIR` hook alongside the
existing `TRACKPAD_FIRMWARE` one: it installs exactly the `brcm/*apple,mriya*`
set into `/lib/firmware/brcm/` and fails if the tree has none. Verified
end-to-end against the staged corpus: 14 files land at the paths above (test
image SHA-256
`4606497f64a80399a10392e8da91f215d9fabb8222e2897f9d447226fc9fbec3`).

## Regeneration recipe (host, ~5 min, ~1.1 GiB network)

```sh
# 1. ranged member fetch (installer checkout provides URLCache)
python3 - <<'PY'
import sys, zipfile, shutil
sys.path.insert(0, "/private/tmp/asahi-installer.cxPxO7/src")
from urlcache import URLCache
url = ("https://updates.cdn-apple.com/2026SpringFCS/fullrestores/140-24263/"
       "B95838F0-6815-4F0B-A039-156526C081AD/UniversalMac_26.5.2_25F84_Restore.ipsw")
zf = zipfile.ZipFile(URLCache(url))
with zf.open("022-21678-099.dmg.aea") as s, open("/private/tmp/t6040-ipsw/022-21678-099.dmg.aea", "wb") as o:
    shutil.copyfileobj(s, o, 1 << 20)
PY
# 2. decrypt + mount
ipsw fw aea /private/tmp/t6040-ipsw/022-21678-099.dmg.aea -o /private/tmp/t6040-ipsw
hdiutil attach -readonly -mountpoint /private/tmp/t6040-basesystem-mnt \
    /private/tmp/t6040-ipsw/022-21678-099.dmg
# 3. copy dext tree resolving symlinks; drop 26.x-unparsable names
rsync -aL --exclude TrustedSHAInformation.txt \
    /private/tmp/t6040-basesystem-mnt/System/Library/DriverExtensions/com.apple.DriverKit-AppleBCMWLAN.dext/Firmware/ \
    /tmp/wifi-src/
find /tmp/wifi-src \( -name "*_F-*" -o -name "*_gen*" \) -type f -delete
# 4. run the unmodified collectors (WiFiFWCollection on /tmp/wifi-src,
#    BluetoothFWCollection on <mnt>/usr/share/firmware/bluetooth) and write
#    each yielded (name, fwfile.data) under vendorfw/ — see this ticket's
#    session script; then hdiutil detach.
```
